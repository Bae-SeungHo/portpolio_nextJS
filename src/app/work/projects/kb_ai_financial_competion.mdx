## 서비스 소개

KB국민은행 고객이 모바일 앱에서 실사 아바타 AI와 대화하며 금융 업무를
처리하는 서비스입니다. 기존 텍스트 챗봇에서 아바타 기반 인터페이스로
전환해 사용자 경험을 크게 개선했습니다.

## 문제

수백만 고객이 이용하는 서비스를 24시간 안정적으로 운영하면서
클라우드 비용도 통제해야 했습니다. 특히 영업 시간 외 트래픽이
급격히 줄어드는데, 서버를 항상 풀로 유지하면 비용 낭비였습니다.
또한 영상 합성 모듈이 CPU/GPU 집약적이라 리소스 관리가 까다로웠습니다.

## 내 역할

- 영상합성 모듈 운영 및 고도화
- Kubernetes 파드 관리 및 리소스 패칭
- Lambda scheduling으로 비사용 시간대 파드 스케일 다운
- HPA(Horizontal Pod Autoscaler) 설정으로 트래픽 급증 자동 대응
- DevOps 및 CI/CD 스크립트 개발
- 학습 데이터 관리 및 서비스 모니터링

## 기술 스택

`Kubernetes` `AWS EKS` `Lambda` `HPA` `Docker` `CI/CD` `Python` `모니터링`

## 해결 방법

Lambda scheduling을 적용해 야간/주말에 불필요한 파드를 자동 종료하고,
평일 오전에 자동으로 다시 기동하도록 설정했습니다.
HPA는 CPU 사용률 70% 기준으로 파드를 2개에서 최대 10개까지 자동 확장합니다.

CI/CD는 GitHub Actions로 구성해 코드 병합 시 자동으로 도커 이미지를 빌드하고
EKS에 롤링 업데이트로 배포합니다. 배포 중 서비스 중단 없이 업데이트가 됩니다.

## 성과

클라우드 비용을 의미 있게 절감했습니다.
배포 프로세스가 자동화돼 운영 부담이 줄었습니다.
모니터링 대시보드로 장애를 사전에 감지하는 체계를 갖췄습니다.

## 느낀 점

"안정성"과 "비용 효율"은 트레이드오프처럼 보이지만, 스케줄링과 자동 스케일링을
잘 조합하면 둘 다 잡을 수 있다는 것을 배웠습니다.
