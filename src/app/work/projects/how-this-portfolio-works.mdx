이 포트폴리오를 처음 기획할 때 스스로에게 두 가지 질문을 던졌습니다.

> "나를 가장 잘 설명하는 방법이 무엇일까?"
> "내가 가진 AI 기술을 포트폴리오 자체에 녹일 수 있을까?"

그 고민의 결과물이 지금 여러분이 보고 계신 이 페이지입니다.

## 기술 스택

| 역할 | 도구 | 선택 이유 |
|------|------|-----------|
| 프레임워크 | Next.js 14 + Once UI | Magic Portfolio 템플릿 기반, 빠른 구성 |
| 배포 | Vercel | git push 한 번으로 글로벌 자동 배포 |
| LLM API | Groq API (Llama 3.3 70B) | 무료, 응답 속도 세계 최고 수준 |
| 벡터 DB | Supabase pgvector | 무료 PostgreSQL + 벡터 검색 |
| 임베딩 | @xenova/transformers (MiniLM) | 완전 무료, 한국어 지원 |
| 코드 관리 | GitHub | 버전 관리 + Vercel 자동 배포 트리거 |

**월 운영 비용: ₩0**. 포트폴리오 규모에서 모든 서비스가 무료 티어로 충분합니다.

## 고민 1 — "포트폴리오를 어떻게 차별화할까?"

일반적인 포트폴리오는 이력서의 웹 버전입니다. 나열하고, 설명하고, 끝.

저는 방문자가 저에 대해 **직접 질문**할 수 있는 경험을 만들고 싶었습니다.
채용담당자가 "Kubernetes 경험이 얼마나 되나요?"라고 궁금해할 때,
페이지를 뒤적이는 게 아니라 바로 물어볼 수 있게요.

그래서 **RAG(검색 증강 생성) 기반 챗봇**을 포트폴리오 안에 내장했습니다.

## 고민 2 — "서버 비용 없이 AI를 어떻게 붙일까?"

처음에는 OpenAI API를 쓰려고 했지만, 비용이 문제였습니다.
포트폴리오는 장기적으로 운영해야 하는데, 방문자마다 비용이 발생하면 곤란합니다.

**해결책: Groq API**. Llama 3.3 70B를 무료로 제공하고,
응답 속도는 오히려 GPT-4보다 빠릅니다.
백엔드는 별도 서버 없이 Next.js API Route로 Vercel에 같이 배포했습니다.

## 고민 3 — "내 정보를 AI가 어떻게 정확히 알게 할까?"

단순히 "당신은 배승호의 어시스턴트입니다"라고만 쓰면 AI가 없는 말을 지어냅니다.
**RAG 파이프라인**을 구축했습니다.

1. 제 이력, 프로젝트, 수상 경력을 14개 청크로 구조화
2. `@xenova/transformers`로 각 청크를 384차원 벡터로 임베딩
3. Supabase pgvector에 저장
4. 질문이 들어오면 코사인 유사도로 가장 관련 있는 청크 4개 검색
5. 검색된 컨텍스트 + 질문을 Groq에 전달 → 정확한 답변 생성

덕분에 "금융AI센터에서 뭘 만들었나요?"처럼 구체적인 질문에도 정확히 답합니다.

## 고민 4 — "한국어 포트폴리오인데 영어로만 쓰는 게 맞나?"

Magic Portfolio 템플릿은 기본적으로 영어입니다.
하지만 저는 국내 채용 담당자와 글로벌 독자 모두를 고려해야 했습니다.

결론: **한국어를 앞에, 영어를 뒤에**.
읽는 사람의 언어에 따라 자연스럽게 선택해서 읽을 수 있도록
두 언어를 하나의 섹션에 조화롭게 배치했습니다.

## 이 포트폴리오가 담고 있는 것

단순히 "이런 기술을 씁니다"가 아니라,
**그 기술로 실제로 무언가를 만들 수 있다는 것**을 보여주고 싶었습니다.
