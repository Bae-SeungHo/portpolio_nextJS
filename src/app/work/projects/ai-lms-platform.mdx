## 문제

KB국민은행에서는 챗봇, 콜봇 등 여러 봇 엔진이 독립적으로 운영되고 있었습니다.
학습 데이터(질의·의도·답변)를 각 엔진마다 별도로 관리해야 했고,
담당자들이 엑셀과 수동 작업으로 처리하는 시간이 상당했습니다.
엔진 간 온톨로지가 달라 통합 분석도 불가능한 상황이었습니다.

## 내 역할과 해결 방법

혼자 전체 백엔드와 인프라를 담당했습니다.

통합 온톨로지 구조를 새로 설계해 여러 봇 엔진의 데이터를 단일 스키마로
수용할 수 있도록 했습니다. Flask REST API로 CRUD와 데이터 분석 엔드포인트를
구현하고, React 프론트엔드와 연동했습니다.

학습 데이터를 임베딩해 유사 질의를 자동으로 클러스터링하고 시각화하는
기능도 추가했습니다. 운영자가 수동으로 중복 질의를 찾던 작업이 자동화됐습니다.

배포는 Docker 이미지로 빌드해 Kubernetes에 올렸습니다.
AWS EKS 위에서 HPA를 적용해 트래픽에 따라 파드가 자동 스케일링됩니다.

## 기술 스택

`Python` `Flask` `React` `MySQL` `Docker` `Kubernetes` `AWS EKS` `NLP` `임베딩`

## 성과

담당자들의 수작업 데이터 관리 시간이 크게 줄었습니다.
여러 봇 엔진 데이터를 한 화면에서 비교·분석할 수 있게 됐습니다.
Kubernetes 무중단 배포로 운영 안정성이 확보됐습니다.
이 프로젝트 경험을 바탕으로 2025년 RDA 에이전트 경진대회에서 우수상을 수상했습니다.

## 느낀 점

단순히 기능을 구현하는 것보다 "운영자가 실제로 쓰기 편한가"를 계속
고민했습니다. 대시보드 UI를 세 번 이상 피드백을 받아 개선했고,
그 결과 실제 사용률이 높아졌습니다.


## Overview

KB Kookmin Bank's chatbot and callbot engines require continuous retraining
with curated Q&A data. This platform centralizes that process — managing
intents, utterances, and responses across multiple bot engines in one place,
while automating what was previously manual QA work.

## My Role

Sole backend engineer for 2+ years. Designed and built the Flask API server,
MySQL schema, Docker containerization, and Kubernetes deployment on AWS.
Also implemented QA data embedding and vector visualization for semantic search.

## Tech Stack

`Python` `Flask` `MySQL` `React` `Kubernetes` `AWS` `Docker` `NLP` `Embeddings`

## Impact

- Automated manual data labeling workflows → reduced ops team workload significantly
- Deployed as a Kubernetes service — high availability, zero-downtime updates
- Used daily by internal teams managing millions of customer interaction intents

## Technologies Used

- **Figma API**: For extracting design tokens and component data directly from the Figma designs.
- **React and Next.js**: For building the front-end codebase with clean, reusable components.
- **Styled-Components**: For managing styles dynamically using design tokens.
- **GitHub Actions**: For automating the pipeline and syncing design changes to the repository.

## Challenges and Learnings

One of the biggest challenges was ensuring that the generated code was clean and maintainable. This involved setting up intelligent mapping between Figma components and React code structures, as well as managing edge cases like responsive design and conditional rendering. Additionally, the continuous synchronization required a robust error-handling system to prevent conflicts during development.

## Outcome

The automated Figma to code pipeline has streamlined the handoff process, cutting down design-to-development time by 40%. Designers now have more confidence that their designs will be accurately translated into code, and developers can focus on more complex logic and feature development. This project has proven the value of automation in bridging the gap between design and development.
