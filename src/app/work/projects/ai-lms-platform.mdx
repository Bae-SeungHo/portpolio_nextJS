## 왜 만들었나

처음엔 "이게 왜 아직 수기야?" 라는 생각에서 시작했습니다.

고객용 챗봇, 직원용 챗봇, 콜봇. 채널마다 담당자가 따로 있고, 각자 로컬 엑셀 파일로 질의-답변 쌍을 관리했습니다. 같은 질문인데 채널마다 다른 답이 나오는 일이 있었고, 신규 상품이 출시되면 모든 담당자가 따로따로 업데이트해야 했습니다.

직접 만들기로 했습니다.

## 개발 과정

처음엔 단순한 Flask 대시보드였습니다. 질의 목록 보여주고 수정하는 정도. 근데 현업 담당자들이 쓰다 보니 요구사항이 계속 생겼습니다.

"시간대별로 어떤 질문이 많이 들어왔는지 볼 수 있어요?" "이 질문이랑 비슷한 질문들 모아서 보고 싶어요."

요구사항을 수렴하면서 FastAPI 기반 백엔드 + HTMX + JavaScript MPA 구조로 전환하고, 질의 유사도 분석을 위한 임베딩 모델도 추가로 학습했습니다.

## 시스템 구조

```text
운영자 브라우저 (HTMX + JS)
    ↓
FastAPI 백엔드
    ├── MySQL (학습 데이터 / 대화 로그)
    └── 임베딩 모델 (질의 유사도 분석)
    ↓
AWS EKS (Kubernetes Deployment)
    └── LB / NAT Gateway / Security Group
```
