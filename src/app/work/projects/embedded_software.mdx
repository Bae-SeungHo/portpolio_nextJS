## 배경

양치질은 매일 하지만 올바른 방법으로 하는 사람은 드뭅니다.
기존 스마트 칫솔은 가격이 비싸고 센서 정확도에 한계가 있었습니다.
카메라 하나로 비접촉 분석이 가능하다면 더 저렴하고 정확한 솔루션이 될 것이라 생각했습니다.

## 팀 구성과 역할 분담

4명이 역할을 나눴습니다.

| 역할 | 담당 |
|------|------|
| 딥러닝 모델 개발 + TCP 통신 | **배승호 (담당)** |
| 임베디드 보드 구현 | 팀원 |
| 웹 서비스 + 시각화 | 팀원 |
| 데이터 수집·분석 | 팀원 |

저는 양치 방향 분류 딥러닝 모델과 보드-서버 간 통신 파이프라인을 맡았습니다.

## 시스템 구조

```
임베디드 보드 (카메라)
↓ TCP 전송
Python 서버 (전처리 · 모델 추론)
↓ 분석 결과 전송
GCP 클라우드
↓
웹 대시보드 (미흡 부위 히트맵 시각화)

```

## 딥러닝 모델

MediaPipe로 손(21개)과 안면(15개) 총 36개 Landmark 좌표를 추출합니다.
현재 프레임과 다음 프레임의 차이를 계산해 손 위치의 학습 의존도를 제거합니다.
1초(60프레임) 단위 Sequential Data를 LSTM 모델에 입력해 양치 방향을 분류합니다.
왼손/오른손 여부를 구분하여 각각 학습된 모델로 추론합니다.

## 기술 스택

`Python` `TensorFlow` `LSTM` `MediaPipe` `OpenCV` `Raspberry Pi` `GCP` `HTML5` `TCP Socket`

## 수상 및 발표

산업통상자원부 주관 제19회 임베디드SW 경진대회 **아크데이타 대표이사상** 수상

대한임베디드공학회 추계학술대회 **우수논문발표상** 수상

고가의 양치 습관 분석 장비보다 저렴하게 구현할 수 있다는 점에서 높은 평가를 받았습니다.

## 느낀 점

4명이 명확하게 역할을 나누고 TCP 소켓으로 임베디드 보드와 서버를 연결하는
실제 협업 경험이었습니다. 하드웨어 제약 안에서 아키텍처를 설계하는 것,
그리고 팀원의 작업물과 내 코드를 인터페이스로 연결하는 것의 어려움과 보람을 배웠습니다.
