## 아이디어

고가의 스마트 칫솔 말고, 카메라 하나로 양치 습관을 분석할 수 있으면 어떨까. 그게 시작이었습니다.

임베디드 보드에 달린 카메라로 사용자가 양치하는 모습을 찍고, 딥러닝 모델이 칫솔 방향과 미흡 부위를 분석해서 웹으로 보여주는 시스템입니다.

## 시스템 흐름

```text
Raspberry Pi (카메라)
    ↓ TCP 소켓 전송
Python 서버 (MediaPipe 전처리 + LSTM 추론)
    ↓ 분석 결과
GCP 클라우드 스토리지
    ↓
웹 대시보드 (양치 점수 + 미흡 부위 히트맵)
```

임베디드 보드에서 직접 추론하지 않고 서버로 영상을 넘기는 방식을 선택했습니다. Raspberry Pi 메모리로는 OpenCV + 딥러닝 모델을 동시에 올리기 어려웠기 때문입니다.

## 딥러닝 모델

MediaPipe로 손(21개)과 얼굴(15개) 총 36개 Landmark 좌표를 추출합니다. 이때 현재 프레임과 다음 프레임의 차이값을 함께 수집합니다. 절대 위치가 아닌 움직임 패턴을 학습하기 위해서입니다.

60프레임(약 1초) 단위 Sequential 데이터를 LSTM에 넣으면 양치 방향(상하/좌우/원형)을 분류합니다. 왼손잡이·오른손잡이를 구분해 각각 다른 모델로 추론합니다.

```python
# LSTM 모델 구조
model = Sequential([
    LSTM(64, return_sequences=False, input_shape=(60, 36*3)),
    Dense(32, activation='relu'),
    Dropout(0.3),
    Dense(4, activation='softmax')  # 양치 방향 4클래스
])
```

## 느낀 점

역할을 명확하게 나눈 협업이 얼마나 중요한지 배웠습니다. 제가 모델을 짜면 팀원이 보드에서 데이터를 흘려보내고, 또 다른 팀원이 웹에서 결과를 받아 시각화하는 — 인터페이스만 잘 정의되면 병렬로 개발할 수 있었습니다. 혼자 다 하는 것보다 훨씬 빨랐습니다.
