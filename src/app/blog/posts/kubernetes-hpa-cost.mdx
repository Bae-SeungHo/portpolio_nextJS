AI 서비스 운영하다 보면 꼭 나오는 질문이 있습니다. "이 서버, 새벽에도 다 켜놔야 해?"

KB국민은행 AI 금융비서 서비스에서 이 문제를 직접 해결했습니다.

## 상황

영업 시간(평일 9시~6시)에 트래픽이 몰리고, 야간·주말엔 거의 없습니다. 파드를 24시간 풀 스펙으로 유지하면 비사용 시간에도 비용이 그대로 나갑니다. 특히 영상 합성 모듈은 CPU/GPU를 많이 잡아먹어서 더 문제였습니다.

## Lambda Scheduling

AWS Lambda로 시간대별 파드 수를 자동 조절했습니다.

```python
# 평일 오전 8시 50분 → 스케일업 (영업 10분 전)
# 평일 오후 6시 10분 → 스케일다운
# 주말 → 최소 유지

def scale_eks_deployment(replicas: int, deployment: str):
    k8s_client.patch_namespaced_deployment_scale(
        name=deployment,
        namespace="production",
        body={"spec": {"replicas": replicas}}
    )
```

## HPA 설정

갑자기 이벤트나 캠페인으로 트래픽이 몰릴 때를 대비해서 HPA도 같이 설정했습니다.

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
spec:
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

평상시엔 Lambda가 비용을 줄이고, 갑자기 몰릴 때는 HPA가 받아줍니다. 70% 기준이나 10분 버퍼 같은 숫자들은 실제 서비스 트래픽 패턴을 보고 정해야 합니다. 이론이 아니라 운영 경험에서 나오는 값들입니다.
