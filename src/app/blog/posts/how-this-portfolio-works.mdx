포트폴리오를 새로 만들면서 딱 한 가지가 욕심났습니다.

방문자가 저에 대해 직접 물어볼 수 있으면 어떨까?

이력서랑 포트폴리오를 아무리 잘 써도, 채용담당자가 원하는 정보를 찾으려면 페이지를 뒤적여야 합니다. 그게 불편해 보였습니다. 그래서 RAG 챗봇을 포트폴리오 안에 넣기로 했습니다.

## 기술 선택

가장 고민됐던 건 LLM API 비용이었습니다. OpenAI를 쓰면 방문자마다 비용이 발생하는데, 개인 포트폴리오에 매달 API 비용을 내는 건 부담입니다.

Groq을 발견했습니다. Llama 3.3 70B를 무료로 제공하고, 응답 속도는 GPT-4보다 빠릅니다. 분당 30 요청, 일 14,400 토큰 — 포트폴리오 트래픽에는 충분합니다.

## RAG가 필요했던 이유

단순히 "당신은 배승호의 어시스턴트입니다"라고만 프롬프트를 쓰면 AI가 모르는 내용을 지어냅니다. 이력서에 없는 경력을 만들어낸다거나, 수상 내역을 틀리게 말한다거나.

RAG는 질문이 들어오면 먼저 지식베이스에서 관련 내용을 검색해서 AI에게 넘겨줍니다. AI가 직접 기억하는 게 아니라, 검색된 정보를 바탕으로 답하는 구조입니다.

```text
방문자 질문
    ↓
질문 → 벡터 변환 (MiniLM 384차원)
    ↓
Supabase pgvector 코사인 유사도 검색
    ↓
관련 청크 4개 추출
    ↓
컨텍스트 + 질문 → Groq API (Llama 3.3 70B)
    ↓
스트리밍 답변 생성
```

실제로 "Kubernetes 운영 경험이 어떻게 되나요?" 같은 구체적인 질문에도 정확하게 답합니다.

## 이 포트폴리오 자체가 하나의 프로젝트

단순히 이력서의 웹 버전을 만들고 싶지 않았습니다. 제가 어떤 걸 할 수 있는지는 포트폴리오를 보면 알 수 있고, 어떻게 생각하는지는 블로그를 보면 알 수 있고, 저에 대해 궁금한 건 챗봇에게 바로 물어볼 수 있는 — 그런 포트폴리오를 만들고 싶었습니다.
