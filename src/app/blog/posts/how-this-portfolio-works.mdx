포트폴리오 다시 만들면서 욕심 하나가 생겼습니다. 방문자가 저에 대해 직접 물어볼 수 있으면 어떨까 싶었습니다.

이력서랑 포트폴리오를 아무리 잘 써도 원하는 정보를 찾으려면 페이지를 뒤적여야 합니다. 그 대신 챗봇에 바로 물어볼 수 있으면 낫겠다 싶어서 RAG 시스템을 직접 붙였습니다.

## 기술 스택

가장 고민됐던 건 LLM API 비용이었습니다. OpenAI를 쓰면 방문자마다 비용이 발생하는데, 개인 포트폴리오에 매달 API 비용을 내는 건 별로였습니다. 그러다 **Groq**을 발견했습니다. Llama 3.3 70B를 무료로 제공하고, 응답 속도는 GPT-4보다 빠릅니다.

## RAG 파이프라인

단순히 프롬프트에 "당신은 배승호의 어시스턴트입니다"라고만 쓰면 AI가 없는 내용을 지어냅니다. RAG는 질문이 들어오면 먼저 지식베이스에서 관련 내용을 검색해서 AI에게 넘겨줍니다.

```text
방문자 질문
    ↓
질문 → 벡터 변환 (MiniLM 384차원)
    ↓
Supabase pgvector 코사인 유사도 검색
    ↓
관련 청크 4개 추출
    ↓
컨텍스트 + 질문 → Groq API (Llama 3.3 70B)
    ↓
스트리밍 답변
```

```typescript
// SSE 스트리밍 응답
const stream = await groq.chat.completions.create({
  model: "llama-3.3-70b-versatile",
  messages: [{ role: "system", content: systemPrompt }, ...messages],
  stream: true,
  temperature: 0.2,
});

for await (const chunk of stream) {
  const text = chunk.choices[0]?.delta?.content || "";
  controller.enqueue(encoder.encode(`data: ${JSON.stringify({ text })}\n\n`));
}
```

이 포트폴리오 자체가 하나의 프로젝트입니다. 단순한 이력서의 웹 버전이 아니라, 직접 만든 AI 시스템이 붙어있는 결과물로 만들고 싶었습니다.
