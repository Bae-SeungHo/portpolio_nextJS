이 포트폴리오를 처음 기획할 때 스스로에게 두 가지 질문을 던졌습니다.

> "나를 가장 잘 설명하는 방법이 무엇일까?"
> "내가 가진 AI 기술을 포트폴리오 자체에 녹일 수 있을까?"

그 고민의 결과물이 지금 여러분이 보고 계신 이 페이지입니다.

이력서와 포트폴리오를 아무리 잘 써도, 원하는 정보를 찾으려면 페이지를 뒤적여야 합니다.
그래서 RAG(검색 증강 생성) 챗봇을 포트폴리오 안에 내장하기로 했습니다.


## 기술 스택 선택 이유

| 역할 | 도구 | 선택 이유 |
|------|------|-----------|
| 프레임워크 | Next.js 14 + Once UI | Magic Portfolio 템플릿 기반 |
| 배포 | Vercel | git push 한 번으로 자동 배포 |
| LLM API | Groq (Llama 3.3 70B) | 무료, 응답 속도 빠름 |
| 벡터 DB | Supabase pgvector | 무료 PostgreSQL + 벡터 검색 |
| 임베딩 | @xenova/transformers | 완전 무료, 한국어 지원 |

**월 운영 비용: ₩0**

## 가장 고민했던 것

처음엔 OpenAI API를 쓰려 했는데, 포트폴리오처럼 장기 운영하는 서비스에
방문자마다 비용이 발생하면 부담입니다. Groq API는 Llama 3.3 70B를
무료로 제공하고, 속도도 GPT-4보다 빠릅니다.

백엔드는 별도 서버 없이 Next.js API Route를 Vercel에 함께 배포했습니다.
프론트와 백엔드가 한 프로젝트에 있으니 관리도 훨씬 단순합니다.

## RAG 파이프라인 동작 방식

1. 제 이력·프로젝트·경력을 14개 텍스트 청크로 구조화
2. `@xenova/transformers`로 각 청크를 384차원 벡터로 변환
3. Supabase pgvector에 저장
4. 방문자 질문 → 코사인 유사도로 관련 청크 4개 검색
5. 검색된 컨텍스트 + 질문 → Groq API → 스트리밍 답변 생성

덕분에 "Kubernetes 경험이 어떻게 되나요?" 같은 구체적인 질문에도 정확히 답합니다.
