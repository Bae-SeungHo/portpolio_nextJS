## 개요

제가 어린시절 아버지께서는 저와 형의 성장 일기를 홈페이지로 만드셨습니다.
사람들이 보고 댓글을 남길 수 있도록 방명록도 구현되어 있었고, 성장 과정을 볼 수 있는 갤러리 페이지도 존재했습니다. (URL이 아직도 기억나는데, bae-family.zc.bz라는 무료 도메인을 사용하셨었네요.)

지금은 사이트가 없지만, 그때의 기억에서인지 저는 항상 저만의 사이트를 만들고 싶었던 마음이 있었습니다.

그리고 대학교 취업준비 중에, Github Pages를 통해 제 첫 포트폴리오 사이트를 배포하였습니다.

## 한 발자국 더

그렇게 시간이 지나 2026년에 이직준비를 하며, 제 정보들을 최신화되고 더 확장된 포트폴리오 사이트를 새로 만들어야 되겠다고 생각이 들어
다른 여러 포트폴리오 레퍼런스들을 찾아 보았습니다.

그러면서 문득 **방문자가 저에 대해 직접 물어볼 수 있으면 어떨까?** 라는 새로운 아이디어가 떠올랐습니다.

이력서랑 포트폴리오를 많이 작성해도 방문자가 보고싶은 정보는 다를 수 있기 때문에,
포트폴리오로서 필요한 정보들을 나열해 놓으면서 동시에 
제 정보를 바탕으로 구축한 RAG 시스템을 통해 궁금한 점을 바로 물어볼 수 있는 시스템을 만들면 좋을 것 같다는 생각이 들었습니다.

## 기술 스택

가장 고민됐던 건 LLM API 비용이었습니다.
OpenAI를 쓰면 답변을 생성할때마다 토큰 비용이 발생하는데, 개인 포트폴리오에 매달 API 비용을 내는 건 별로였습니다.
그러다가 Llama 3.3 70B를 무료로 제공하고, 응답 속도는 GPT-4보다 빠른 **Groq**을 발견했는데, 적당한 사용량에 대해 무료 요금제를 제공해서,
딱 제 포트폴리오에 적합한 LLM으로 사용하게 되었습니다. 

그리고, 서버 배포도 Vercel을 통해 무료로 자동화하여 배포하였고, 방명록 등의 정보를 담는 DB는 Supabase를 통해 연결했습니다.

## RAG 파이프라인

단순히 프롬프트에 "당신은 배승호의 어시스턴트입니다"라고만 쓰면 AI가 없는 내용을 지어냅니다.
Supabase DB에 저에 관련된 내용을 임베딩하여 지식베이스를 구축하고 Context를 생성하여 AI에게 넘겨줍니다.

```text
방문자 질문
    ↓
질문 → 벡터 변환 (MiniLM 384차원)
    ↓
Supabase pgvector 코사인 유사도 검색
    ↓
관련 청크 4개 추출
    ↓
컨텍스트 + 질문 → Groq API (Llama 3.3 70B)
    ↓
스트리밍 답변
```
![supabase](/blogs/supabase.png)


```typescript
// SSE 스트리밍 응답
const stream = await groq.chat.completions.create({
  model: "llama-3.3-70b-versatile",
  messages: [{ role: "system", content: systemPrompt }, ...messages],
  stream: true,
  temperature: 0.2,
});

for await (const chunk of stream) {
  const text = chunk.choices[0]?.delta?.content || "";
  controller.enqueue(encoder.encode(`data: ${JSON.stringify({ text })}\n\n`));
}
```

비용이 하나도 들지 않는 저만의 사이트를 만든 것 같아 매우 뿌듯합니다.
