AI 서비스를 운영하다 보면 반드시 마주치는 문제가 있습니다.

"이 서버, 새벽에도 켜놔야 해?"

KB국민은행 AI 금융비서 서비스를 운영하면서 이 문제를 직접 해결했습니다.

## 상황

영업 시간(평일 오전 9시 ~ 오후 6시)에 트래픽이 집중되고,
야간과 주말에는 거의 사용이 없습니다.
그런데 Kubernetes 파드를 풀 스펙으로 24시간 유지하면
비사용 시간대에도 비용이 그대로 발생합니다.

특히 영상 합성 모듈은 CPU/GPU를 많이 쓰는 무거운 컨테이너라
더욱 비효율적이었습니다.

## 해결: Lambda Scheduling

AWS Lambda를 활용해 시간대별 파드 수를 자동 조절했습니다.

```python
# 평일 오전 8시 50분 → 파드 스케일업
# 평일 오후 6시 10분 → 파드 스케일다운
# 주말 → 최소 유지

import boto3

def scale_deployment(replicas: int):
    # EKS 클러스터의 deployment replicas 조정
    ...
```

영업 시작 10분 전에 미리 파드를 올려두고,
영업 종료 후 10분 뒤에 내리는 방식입니다.
콜드 스타트 시간을 고려한 버퍼입니다.

## HPA(Horizontal Pod Autoscaler)

예외 상황 — 이벤트나 캠페인으로 트래픽이 갑자기 몰리는 경우를 대비해
HPA도 함께 설정했습니다.

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
spec:
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
```

CPU 사용률 70% 기준으로 파드를 2개에서 최대 10개까지 자동 확장합니다.

## 결과

비사용 시간대 클라우드 비용을 의미 있게 절감했습니다.
트래픽 급증 시에도 HPA가 자동으로 대응해 장애 없이 운영됐습니다.

## 느낀 점

Kubernetes 운영은 설정 파일 몇 줄이지만,
그 뒤에 있는 판단 — 언제 올리고 언제 내릴지, 얼마나 버퍼를 둘지 —
이 부분이 실제 운영 경험에서 나온다는 걸 느꼈습니다.
