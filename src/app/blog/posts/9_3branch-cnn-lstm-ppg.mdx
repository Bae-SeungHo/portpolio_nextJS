2022년 여름, ETRI(한국전자통신연구원)에서 두 달간 인턴을 하면서
PPG 신호로 혈압을 추정하는 딥러닝 모델을 개발했습니다.

두 달이라는 짧은 기간이었지만, 의료 AI에서 배운 것들이 지금도 많이 남아있습니다.

## PPG가 뭐고 왜 혈압 추정에 쓰나

PPG(광용적맥파)는 손가락이나 손목에 광센서를 대면
빛의 흡수량 변화로 맥박을 측정하는 신호입니다.
스마트워치의 심박수 측정이 이 원리를 씁니다.

혈압을 측정하려면 보통 커프로 팔을 조입니다.
연속 측정이 불가능하고, 웨어러블에 넣기도 어렵습니다.
PPG는 비침습적으로 연속 측정할 수 있어서
혈압 추정에 활용하려는 연구가 많습니다.

## 데이터: MIMIC-III

MIMIC-III는 Beth Israel Deaconess Medical Center 중환자실 데이터입니다.
약 1,600명 환자의 생체신호 데이터가 공개되어 있습니다.

8초 이상 연속 데이터를 환자당 최대 100개씩 추출하고,
Peak noise와 Flat line을 제거하는 전처리 후 101,087개를 확보했습니다.

데이터 규모는 충분했지만, 병원 ICU 환경이라는 도메인 특성이 있어서
일반 생활 환경 데이터와 분포가 다를 수 있다는 한계가 있었습니다.

## 3-Branch 구조

처음에는 단순 LSTM으로 시작했습니다.
PPG 파형을 시계열로 넣고 SBP를 예측하는 구조입니다.

문제는 PPG 신호에서 뽑을 수 있는 정보가 두 종류라는 점입니다.

첫째는 파형 자체의 형태(Morphology)입니다.
수축기 피크의 높이, 이완기 트로프 위치, 노치 모양 같은 것들입니다.

둘째는 Time-Domain 특성입니다.
피크-투-피크 간격(맥박 주기), 상승 시간, 하강 시간 같은 시간 영역 지표입니다.

이 두 종류의 정보를 하나의 Sequential 입력으로 넣으면
모델이 두 특성을 동등하게 처리하기 어렵습니다.

그래서 Branch를 나눴습니다.

Branch 1과 2는 CNN-LSTM으로 파형 형태를 처리하고
(수용 영역 크기만 다르게 설정),
Branch 3은 DNN으로 Time-Domain 특성을 처리합니다.
세 Branch의 출력을 합쳐서 최종 SBP를 예측합니다.

```
[PPG 신호] → Branch 1 (General CNN-LSTM) →┐
[PPG 신호] → Branch 2 (Wide CNN-LSTM)   →├→ Dense → SBP
[TD 특성] → Branch 3 (Time-domain DNN) →┘

```

## 결과

단순 CNN-LSTM만 쓸 때 MAE가 9.98 mmHg였는데,
세 Branch를 모두 쓰면 8.92 mmHg로 약 12% 개선됐습니다.

Time-Domain Branch가 추가됐을 때 개선 폭이 가장 컸습니다.
PPG 파형의 형태 정보만으로는 놓치는 혈압 관련 특성이
시간 영역에 담겨있다는 의미입니다.

## 배운 것

모델 구조보다 데이터 이해가 먼저라는 걸 배웠습니다.

PPG 신호에서 뽑을 수 있는 정보의 종류를 먼저 파악하고,
그에 맞는 구조를 설계하니 성능이 올랐습니다.
```하나의 입력, 하나의 모델```이라는 단순한 생각에서 벗어나는 게 핵심이었습니다.

논문으로 한국통신학회에 발표했습니다.
짧은 인턴 기간에 연구 논문 발표까지 마친 경험이 이후 연구 활동에 자신감이 됐습니다.
