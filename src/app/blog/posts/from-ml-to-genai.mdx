"LLM으로 바꾸면 더 잘 되지 않나요?"

이 질문을 받을 때마다 "그렇기도 하고 아니기도 하다"고 답합니다.

KB국민은행에서 기존 ML 기반 챗봇을 생성형 AI 아키텍처로 전환하는 작업을
직접 진행하면서 느낀 것들을 정리합니다.

## 기존 ML 챗봇의 구조

기존 챗봇은 의도 분류(Intent Classification) + 개체 추출(NER) 방식이었습니다.

사용자 입력 → 의도 분류 모델 → 의도 매핑 → 답변 템플릿 반환


장점이 뚜렷합니다.
- 예측 가능하다 — 같은 입력에 같은 답이 나온다
- 빠르다 — 추론 시간이 짧다
- 통제하기 쉽다 — 답변이 템플릿 범위를 벗어나지 않는다

## LLM으로 전환할 때 생기는 문제들

### 1. 할루시네이션

LLM은 모르는 것을 지어냅니다.
금융 도메인에서 틀린 정보를 자신 있게 말하면 심각한 문제가 됩니다.

RAG로 어느 정도 해결되지만, 완벽하지 않습니다.
시스템 프롬프트에 "모르면 모른다고 말하라"를 명시하고,
검색 유사도 임계값 이하면 답변 자체를 막는 방식으로 보완했습니다.

### 2. 일관성 없는 답변

같은 질문에 매번 다른 표현으로 답합니다.
temperature를 낮추면 줄어들지만 완전히 없앨 수는 없습니다.
품질 모니터링 체계를 새로 만들어야 했습니다.

### 3. 속도

ML 모델 추론은 수십 ms지만 LLM은 수백 ms~수 초입니다.
스트리밍 응답으로 체감 속도를 개선했지만,
첫 토큰까지의 시간(TTFT)은 여전히 신경 써야 합니다.

### 4. 비용

추론 비용이 기존 ML보다 훨씬 높습니다.
캐싱 전략(자주 나오는 질문은 결과 캐시)을 적용해 비용을 줄였습니다.

## 그래서 전환할 가치가 있나요?

의도 분류 기반 챗봇은 커버하지 못하는 케이스가 항상 존재합니다.
학습 데이터에 없는 질문은 "죄송합니다, 이해하지 못했습니다"로 끝납니다.

LLM 기반 챗봇은 이 케이스를 자연스럽게 처리합니다.
한 번도 학습하지 않은 표현도 이해하고, 맥락을 이어서 대화합니다.

결국 트레이드오프입니다.
예측 가능성과 비용이 중요하면 ML,
자연스러운 대화와 커버리지가 중요하면 LLM입니다.
두 가지를 적절히 섞는 하이브리드 방식도 고려해볼 만합니다.
