오픈소스 기여는 오랫동안 "언젠가 해야지"로 미뤄온 것 중 하나였습니다.

막연히 어려울 것 같았습니다.
유명한 레포에 코드를 올리면 전문가들이 지적하지 않을까,
영어로 소통해야 하는데 잘 될까, 같은 걱정이 있었습니다.

그러다 올해 실제로 해봤습니다.

## 어디서 시작했나

처음부터 큰 레포를 노린 건 아닙니다.

평소 쓰는 도구 중에 한국어 지원이 아쉬운 것이 있었습니다.
Flowise의 UI 번역 파일을 보니 한국어 항목이 빠진 게 꽤 있었습니다.

이건 할 수 있겠다 싶었습니다.

## 과정

```bash
# 1. 레포 Fork
gh repo fork FlowiseAI/Flowise --clone

# 2. 브랜치 생성
git checkout -b fix/korean-i18n-improvements

# 3. 번역 파일 수정
# packages/ui/src/i18n/locales/ko.json
# 빠진 항목들 추가

# 4. 커밋 & 푸시
git commit -m "fix(i18n): add missing Korean translations for chatflow"
git push origin fix/korean-i18n-improvements

# 5. GitHub에서 PR 오픈
```

PR을 올리고 나서 두 가지 피드백을 받았습니다.
하나는 번역 표현을 좀 더 자연스럽게 바꿔달라는 것,
하나는 다른 파일에도 같은 패턴으로 적용해달라는 것이었습니다.

수정해서 올렸고, 머지됐습니다.

## 생각보다 괜찮았던 것들

리뷰어들이 생각보다 친절했습니다.
틀린 걸 지적할 때도 "이렇게 하면 어떨까요?"라는 식으로 제안했습니다.

영어 소통도 생각만큼 부담스럽지 않았습니다.
PR 설명과 댓글은 짧고 명확하게 쓰면 됩니다.

## 다음 목표

이번엔 번역이었지만, 다음엔 코드 수준의 기여를 해보고 싶습니다.

개발자로서 중요한 역량이라는 걸 이번에 느꼈습니다.
