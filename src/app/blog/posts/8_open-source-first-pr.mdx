오픈소스 기여는 오랫동안 "언젠가 해야지"로 미뤄온 목록 1위였습니다.

막연히 어려울 것 같았습니다. 유명한 레포에 코드를 올리면 전문가들이 지적하지 않을까. 영어로 소통해야 하는데 잘 될까.

그러다 올해 실제로 해봤습니다.

## 어디서 시작했나

처음부터 큰 레포를 노린 건 아닙니다.

평소에 쓰는 Flowise를 보다가 한국어 번역 파일에 빠진 항목들이 꽤 있었습니다. 이건 할 수 있겠다 싶었습니다.

## 과정

```bash
# 1. 레포 Fork & Clone
gh repo fork FlowiseAI/Flowise --clone
cd Flowise

# 2. 브랜치 생성
git checkout -b fix/korean-i18n-improvements

# 3. 번역 파일 수정
# packages/ui/src/i18n/locales/ko.json
# 빠진 항목 추가, 어색한 표현 수정

# 4. 커밋 & 푸시
git commit -m "fix(i18n): add missing Korean translations for chatflow and agentflow"
git push origin fix/korean-i18n-improvements

# 5. GitHub에서 PR 오픈
```

PR을 올리고 두 가지 피드백을 받았습니다. 번역 표현을 좀 더 자연스럽게 바꿔달라는 것, 다른 파일에도 같은 패턴으로 적용해달라는 것. 수정해서 올렸고 머지됐습니다.

## 생각보다 괜찮았던 것들

리뷰어들이 생각보다 친절했습니다. 틀린 걸 지적할 때도 "이렇게 하면 어떨까요?" 식이었습니다.

영어 소통도 부담스럽지 않았습니다. PR 설명과 댓글은 짧고 명확하게 쓰면 됩니다.

코드 한 줄 안 건드리고 번역 파일만 수정했는데도 Contributors 목록에 이름이 올라갔습니다.

## 다음 목표

번역이었지만, 다음엔 코드 수준의 기여를 해보고 싶습니다. LangChain 한국어 예제 추가나, 사용하면서 발견한 버그 수정 같은 것들을요.

"좋은 생태계에 기여하는 것"도 개발자로서 중요한 능력이라는 걸 이번에 느꼈습니다.
