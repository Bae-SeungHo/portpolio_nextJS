반복하는 업무 중에 "이거 AI가 하면 되겠는데"라는 생각이 드는 순간이 있습니다.

저에게는 기술 뉴스를 읽고 정리하는 작업이 그랬습니다.
매일 GeekNews, YozmIT, GitHub Trending을 훑어보고
관심 있는 내용을 메모하는데, 이 과정 자체가 시간이 꽤 걸립니다.

그래서 이걸 자동화해봤습니다.

## 구성

Crawl4AI (크롤링) → Ollama (요약·선별) → n8n (스케줄·발행)

**Crawl4AI**로 여러 기술 뉴스 사이트를 크롤링하고,
**Ollama**의 Qwen2.5:32b 모델로 관련성과 중요도를 스코어링합니다.
**n8n**이 매일 오전 스케줄러를 돌려서 전체 파이프라인을 실행합니다.

## 실제로 써보니

AI가 선별한 뉴스 품질이 생각보다 괜찮았습니다.
특히 GitHub Trending에서 건져온 레포들 중에
평소엔 못 봤을 것들이 들어오는 게 좋았습니다.

반면 아직 아쉬운 것도 있습니다.
맥락 없이 인기가 높다고 선별하는 경우가 있어서,
저의 관심 분야(AI·MLOps·Kubernetes)에 특화된 필터를 추가 중입니다.

## 회사 A5000 GPU 활용

회사 PC에 NVIDIA A5000 GPU가 있어서,
Ollama로 70B 수준의 모델도 로컬에서 구동할 수 있습니다.
API 비용 없이 실험할 수 있다는 게 큰 장점입니다.

## 다음 단계

good-first-issue를 찾아주는 에이전트를 만들어볼 생각입니다.
